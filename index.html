Git と GitHub

今日覚えて欲しい事

  1. そもそもGit と GitHubって？？
    名前が似てる様で全く違うもの。。。
    • Git
      • 自分たちが制作したソースコードの管理を行うもの。
        変更内容や変更履歴を管理してくれる。
        全てコマンドを使って操作する。
        Gitが持っているサーバーを利用して管理します。
    • GitHub
      • リモートサーバーの中身をWeb上で見ることができる便利なツール
        Gitのコマンドを使ってリモートサーバーにリポジトリを作ることもできるが！
        GitHubを使ってもっと簡単にリモートリポジトリを作れる！
  2. 覚えて欲しい項目
    • リポジトリ

       リポジトリとは
        ソースコード等を保管してくれる保管庫、保存場所。
        ここからは銀行を例にしていきます！
        ではでは本題。リポジトリにも下記２つのリポジトリがある。

      1. リモートリポジトリ
        • Gitが持っているサーバー上の保管庫。
          →銀行のATMや金庫の様にお金を預けているところ。ソースコードを預け入れます！
      2. ローカルリポジトリ
        • 皆さん自身が扱っているパソコンのことWindowsやMacなど。
          →自宅のタンス預金、自宅の金庫。
    • GitやGitHubで必要になる言葉と意味
      1. クローン(clone)
        • リモートリポジトリのファイルを新規でローカルリポジトリに複製させる処理。
          →銀行からお金をおろして、自宅の金庫に収める。
      2. プル(pull)
        • リモートリポジトリに存在する、修正、変更されたファイルをローカルに引っ張ってくる。
          →通帳に記帳するイメージ。
      3. ブランチ(branch)
        • リモートリポジトリにはメインとなるフォルダがあり、それをリリース版にする事が前提。
          そのメインとなるフォルダから別に枝分かれさせて新しい機能や修正をする。「ブランチを切る」と言います。
          ※ブランチを切り、枝分かれ先で変更、修正してもメイン、リリースされている物には影響なし！！
          →「普通預金」がメインの口座。定期預金や外貨預金にブランチを切る。そこでいくら定期的に預けるかなど思案する。
      4. コミット(commit)
        • 変更、修正したファイルをローカルリポジトリに記録や保存、アップロードする事。
          →臨時収入があってそのお金を一旦、自宅の金庫に保管。
      5. プッシュ(push)
        • ローカルリポジトリからリモートリポジトリにアップロードする事。
          →一旦保管しておいたお金を普通預金や定期預金等に預ける。
      6. マージ(merge)
        • ブランチを切って作業したファイルをメインとなるコードに加える事。
          →定期預金や外貨預金のお金を全て普通預金に移動させる。
